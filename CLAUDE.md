# StockChain MVP - 심플한 발주 등록 앱

## 프로젝트 개요
**목적**: 기존 마켓봄 대비 비용 절감을 위한 심플한 자체 발주 관리 시스템
**타겟**: 식당 운영자들의 간편한 발주 관리
**플랫폼**: 웹/모바일 겸용 (반응형)
**핵심 가치**: **심플함, 실용성, 빠른 처리**

## 기술 스택
- **Frontend**: React 18 + TypeScript + Vite
- **UI 라이브러리**: ShadCN/UI + Tailwind CSS
- **상태 관리**: React Query
- **라우팅**: React Router DOM
- **차트**: Recharts (통계용)
- **폼**: React Hook Form + Zod
- **데이터**: 로컬 스토리지 (MVP) → 향후 DB 연동

## 사양서 vs 마켓봄 비교 분석

### 🎯 사양서 요구사항 (우리 목표)
- **심플한 UI**: "쓸데없는 게 많아서 정리"
- **빠른 주문 등록**: 간단하게 주문 등록
- **월별 필터링**: 이번달/저번달/저저번달 버튼
- **CSV 내보내기**: 데이터 export 기능
- **마켓봄 스타일 통계**: "배껴서 넣으셈"

### 📊 마켓봄의 복잡함 (우리가 피할 것)
- 너무 많은 필터 옵션 (10개 이상)
- 복잡한 주문 프로세스 (담당자, 배송유형 등)
- 과도한 정보 밀도
- 어려운 네비게이션 구조

## 핵심 페이지 구조

### 1. 메인 페이지 (4개)
- **대시보드** (`/`) - 오늘 주문 + 간단 주문 등록
- **주문등록** (`/order`) - 간단한 주문 등록 페이지
- **주문목록** (`/orders`) - 필터링 + CSV 내보내기
- **통계** (`/stats`) - 마켓봄 스타일 통계

### 2. 서브 페이지 (인증/설정)
- **로그인** (`/login`) - 아이디/비밀번호
- **회원가입** (`/signup`) - 기본 정보 + 매장 정보
- **매장선택** (`/stores`) - 현재 화면 유지
- **매장등록** (`/store/new`) - 매장 추가

## 상세 기능 명세

### 🏠 대시보드 (메인 화면 개선)
**현재 문제**: "지금 화면 좀 쓸데없는게 많아서 정리좀"

**개선 방향**:
- 오늘 주문 요약 (건수, 금액)
- 간단 주문 등록 (빠른 액세스)
- 최근 주문 3-5개 목록
- 재고 알림 (부족한 품목)

**제거할 요소**:
- 복잡한 카드들
- 과도한 네비게이션
- 불필요한 통계

### 📝 주문등록 (심플화)
**마켓봄과의 차이점**:
- 거래처 선택 → **매장 내 간편 입력**
- 담당자 지정 → **제거**
- 배송유형 → **제거**
- 복잡한 상품 정보 → **간소화**

**입력 필드**:
```
품목명 → 수량 → 단가 → (자동계산: 총액)
```

**핵심 기능**:
- 자동완성 (이전 주문 기반)
- 실시간 총액 계산
- 빠른 저장
- 즉시 목록 이동

### 📋 주문목록 (월별 필터 중심)
**사양서 요구사항**:
- "이번달/저번달/저저번달 버튼" - 월별 빠른 접근
- "CSV파일로 내보내기 기능"

**필터링 (심플화)**:
- 월별 버튼: [이번달] [저번달] [저저번달] [직접선택]
- 키워드 검색: 품목명 검색
- 상태: 전체/완료/취소

**마켓봄 대비 제거**:
- 복잡한 날짜 검색 (라디오 버튼들)
- 담당자 필터
- 배송 상태 필터
- 브랜드 필터

### 📊 통계 (마켓봄 스타일)
**사양서 요구사항**: "마켓봄 참고해서 배껴서 넣으셈"

**마켓봄에서 가져올 요소**:
- 주문 현황 박스 (당일/월간 주문건수, 금액)
- 깔끔한 카드 레이아웃
- 색상 코딩 (완료/대기/취소)

**추가 요구사항**:
- "일별 주문량 그래프 (한달치)"

**구성**:
```
[주문 현황]     [월간 요약]
당일: X건 XX원   이번달: X건 XX원
                저번달: X건 XX원

[일별 주문량 그래프 - 30일]
📈 Recharts 라인 차트
```

## 데이터 모델 (심플화)

### 주문 (Orders)
```typescript
interface Order {
  id: string;              // 주문번호 (자동생성)
  storeId: string;         // 매장 ID
  items: OrderItem[];      // 주문 상품
  totalAmount: number;     // 총 금액
  status: 'completed' | 'cancelled';  // 상태 (단순화)
  orderDate: Date;         // 주문일시
  createdAt: Date;
}
```

### 주문 상품 (OrderItems)
```typescript
interface OrderItem {
  id: string;
  productName: string;     // 품목명
  quantity: number;        // 수량
  unitPrice: number;       // 단가
  totalPrice: number;      // 총액 (수량 × 단가)
}
```

### 매장 (Stores)
```typescript
interface Store {
  id: string;
  name: string;            // 매장명
  address: string;         // 주소
  managerPhone: string;    // 관리자 전화번호
  createdAt: Date;
}
```

### 사용자 (Users)
```typescript
interface User {
  id: string;
  username: string;        // 아이디
  password: string;        // 비밀번호 (해시)
  stores: string[];        // 관리 매장 목록
  createdAt: Date;
}
```

### 디자인 철학: "Less is More"
- **최소한의 클릭**: 3클릭 이내 모든 작업 완료
- **큰 터치 영역**: 모바일 친화적

- **빠른 피드백**: 즉시 결과 표시

Primary: #2563eb     /* 파란색 - 주요 액션 */
